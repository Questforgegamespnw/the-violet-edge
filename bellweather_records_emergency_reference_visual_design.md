# Bellweather Records & Emergency Reference
## Visual Design Direction for Wiki and Print

> **Core design premise:** A real-world campaign reference presented as though Bellweather assembled it for its own survival.

> **Working visual phrase:** A municipal records office that has begun answering back.

> **Primary title direction:** **Bellweather Records & Emergency Reference**

> **Supporting line:** Compiled for civic continuity, field use, and the preservation of Bellweather under abnormal conditions.

---

# 1. Design Goal

The Bellweather wiki should not look like a conventional fantasy campaign dashboard with a sepia filter applied.

It should establish its own visual language: halfway between an in-world civic artifact and a practical meta-level campaign reference.

The site must feel believable enough to have been produced by Bellweather itself while remaining clear, fast, searchable, and useful at the table.

The design should evoke:

- a county emergency binder
- a municipal archive
- a local newspaper reference file
- a town council records cabinet
- a hospital or utility procedure manual
- a public library reference binder
- an investigator's active case file

The interface should feel competent, civic, workmanlike, and almost reassuring.

The horror enters when one thing is wrong.

---

# 2. Core Identity

The strongest framing is:

> **A real-world reference site presented as though Bellweather assembled it for its own survival.**

This permits a useful balance between immersion and usability.

The wiki does not need to pretend every paragraph is literally an in-world document. Instead, the full interface carries the visual and organizational logic of Bellweather's emergency records system.

## In-World Surface

The following elements should feel native to Bellweather:

- headings
- stamps
- filing labels
- tabs
- warnings
- record codes
- section numbers
- public notices
- archive markers
- revision stamps
- civic seals
- confidentiality classifications

Possible labels include:

- `EMERGENCY REFERENCE COPY`
- `PUBLIC DISTRIBUTION`
- `KEEPER FILE`
- `RECORD VERIFIED`
- `ROUTE STATUS UNKNOWN`
- `DO NOT REPRINT WITHOUT REVIEW`
- `CURRENT CIVIC GUIDANCE`
- `ARCHIVED BEFORE DISPLACEMENT`
- `FIELD USE`
- `LAST WITNESSED`
- `REVISION PENDING`

## Meta-Level Structure

The site itself should still behave like a strong modern wiki.

Preserve:

- readable typography
- responsive navigation
- full-text search
- cross-references
- spoiler separation
- player-safe and GM-only filtering
- quick-reference links
- clear breadcrumbs
- stable page hierarchy
- useful related-record links

The artifact should feel immersive without becoming inconvenient.

---

# 3. Visual North Star

> Bellweather should look ordinary enough to mourn, old enough to remember, and strange enough to make every sign, lamp, road, and record feel like it might answer back.

The visual hierarchy should follow the setting's core dramatic rule:

> **Ordinary first. Uncanny second.**

The default interface should not look haunted.

The uncanny should appear as contamination, revision, duplication, overcorrection, or an impossible addition inside an otherwise trustworthy civic system.

---

# 4. Three Visual Layers

## 4.1 Civic Mode

This is the default visual mode for most pages.

Use for:

- town overview
- daily life
- resources
- civic systems
- public-facing rules
- ordinary locations
- institutions
- player-safe reference pages
- session procedures
- practical campaign guidance

### Visual Character

- warm cream paper
- charcoal or iron-black text
- muted navy and olive headings
- thin ruled borders
- brass or rust accents
- square or lightly rounded corners
- restrained shadows
- modest document texture
- minimal visible Violet influence

### Emotional Effect

This mode should feel:

- practical
- trustworthy
- civic
- local
- organized
- slightly worn
- built for repeated use

---

## 4.2 Archive Mode

Use for:

- Bellweather history
- civic memory
- old roads
- old boundary sites
- saints and local traditions
- family records
- founding documents
- antique maps
- historical notices
- cemetery records
- forgotten institutions

### Visual Character

- darker parchment or ivory
- faded teal or old blue accents
- antique gold and oxidized brass
- engraved or civic-allegorical ornaments
- formal serif headings
- mounted-photo frames
- archive tags
- marginal notes
- date stamps
- document provenance labels

### Emotional Effect

This mode should feel:

- old
- remembered
- ceremonial
- sincere
- civic-mythic
- haunted by continuity rather than decay

---

## 4.3 Violet Mode

Use sparingly for:

- dreams
- manifestations
- altered records
- Violet logic
- damaged routes
- unstable identities
- interpretation matrices
- supplied infrastructure
- pages whose meaning has begun to drift

### Visual Character

- faint violet-gray edge glow
- doubled or forked rules
- soft lavender-gray stains
- nearly imperceptible text shadow
- slight alignment errors
- duplicated labels
- over-precise metadata
- translucent fog at section edges
- indigo-black emphasis blocks
- annotations no clerk remembers adding

### Emotional Effect

The Violet should feel:

- beautiful but wrong
- helpful before frightening
- precise in the wrong places
- emotionally convincing
- patient
- restrained
- contextless in its care

Avoid neon purple, generic magical glow, slime, or overt cosmic ornament.

---

# 5. Relationship to Call of Cthulhu

The design may borrow structural instincts from Call of Cthulhu publications without imitating their layout directly.

Useful influences include:

- cream paper surfaces
- dark restrained typography
- strong chapter-title hierarchy
- framed callout boxes
- thin ornamental rules
- occasional full-width mood art
- clear distinctions between Keeper notes, examples, procedures, and handouts
- generous space around major headings

The crucial distinction is:

> Call of Cthulhu often looks like an old forbidden book.

> Bellweather should look like a county emergency binder that has become cosmologically important.

Bellweather should be less ornate, less overtly occult, and more institutional.

---

# 6. Typography

Use three distinct typographic roles.

## 6.1 Display Headings

Use a tall condensed sans or restrained geometric display face.

It should evoke:

- newspaper mastheads
- civic posters
- public notices
- rail timetables
- WPA publications
- municipal signage

Use for:

- chapter titles
- section headers
- active incident names
- major public notices
- binder dividers

Avoid overly decorative occult display fonts.

## 6.2 Body Text

Use a highly readable book serif.

The body type should feel like:

- a municipal report
- a quality newspaper
- a county reference book
- a professional historical publication

It should not feel like:

- a medieval grimoire
- a distressed typewriter page
- a handwritten journal
- an antique novelty font

## 6.3 Labels and Metadata

Use a narrow sans or restrained monospaced face for:

- `PLAYER SAFE`
- `KEEPER ONLY`
- `FILED UNDER`
- `RELATED RECORDS`
- `TABLE USE`
- `STATUS`
- `LAST VERIFIED`
- `LAST WITNESSED`
- route numbers
- incident codes
- tracker values
- dates and revision notes

This allows the manuscript metadata to become visible interface language.

- 06 Faith, Dreams & Meaning
- 07 Persons & Institutions
- 08 Locations
- 09 Manifestations

## Search

The search field may be labeled:

- `SEARCH RECORDS`
- `CHECK THE INDEX`
- `LOCATE A FILE`
- `REFERENCE LOOKUP`

Search must remain obvious and functional despite the in-world styling.

# 8. Standard Page Anatomy

## Record Header

Each page may display:

- section number
- page title
- record classification
- player-safe or Keeper-only status
- table-use rating
- document status
- last verified date
- related records
- record ID

## Opening Summary

Use an `AT A GLANCE` block styled like a typed index card clipped to the file.

Possible fields:

- Purpose
- What Players Know
- What the GM Knows
- Table Use
- Current Risk
- Related Procedures

## Main Article

Use readable serif text with:

- moderate line length
- generous paragraph spacing
- clear heading hierarchy
- limited decorative interruption
- visible cross-links

## Margin or Inline Notices

Use consistent callout families:

- Keeper's Note
- Public Knowledge
- Violet Misread
- Civic Action
- What the Players See
- What the GM Knows
- What Can Go Wrong
- At the Table
- Related Record
- Current Procedure

## Footer

Include:

- previous record
- next record
- related records
- record ID
- page status
- Bellweather seal or filing stamp

# 9. Component Styles

## 9.1 Keeper's Note

Visual treatment:

- charcoal or dark olive label
- cream body
- square corners
- file-tab header
- restrained confidentiality stamp

Tone:

- operational
- direct
- clearly separate from player-facing text

## 9.2 Public Knowledge

Visual treatment:

- faded navy border
- municipal seal icon
- lighter paper surface
- clean, trustworthy appearance

## 9.3 Violet Misread

Visual treatment:

- pale violet-gray surface
- doubled border
- slight misalignment
- subtly abnormal spacing
- restrained background fog

The component should appear almost normal at first glance.

## 9.4 Civic Action

Visual treatment:

- rust or brass rule
- checklist or procedural layout
- numbered steps
- visible consequences or tracker effects

## 9.5 What the Players See

Visual treatment:

- field report
- local newspaper column
- witness statement
- scene-framing card

## 9.6 What the GM Knows

Visual treatment:

- confidential memorandum
- `KEEPER FILE` stamp
- dark header band
- clearer spoiler separation

## 9.7 Trouble Webs

Style as active incident files.

Possible fields:

- incident title
- surface problem
- Violet misread
- actual logic
- current pressure
- involved persons
- affected locations
- escalation
- possible interventions
- what the Violet may learn

## 9.8 NPCs

Style as personnel records.

Possible fields:

- name
- occupation
- public role
- private pressure
- affiliations
- current status
- last known location
- relevant incidents

## 9.9 Locations

Style as survey or site files.

Possible fields:

- civic district
- function
- current condition
- responsible institution
- known routes
- witness status
- Violet risk
- related incidents

## 9.10 Monsters and Manifestations

Style as abnormal-condition reports rather than a conventional bestiary.

Possible labels:

- observed form
- likely source
- repeated role
- witness consistency
- containment guidance
- recognition test
- interpretive risk
- do not name as

# 13. Background and Texture

Do not over-texture the wiki.

Use:

- nearly flat warm paper
- extremely subtle grain
- dark outer canvas behind the content surface
- restrained vignette on chapter openers
- modest wear on archival pages
- occasional meaningful stains or overwritten marks

Avoid:

- torn edges on every page
- constant coffee stains
- heavy grunge
- aggressive paper shadows
- distressed body text
- decorative damage without narrative purpose

Corruption should be meaningful.

A normal page may be clean.

A page about a rewritten ledger may contain:

- an overwritten name
- a Violet underprint
- a duplicated rule
- an impossible revision date
- an annotation no clerk remembers adding

---

# 14. Print Translation

The same visual system should translate into the printed book.

## Main Book

The book should use the full visual language:

- civic chapter openers
- archive pages
- incident files
- personnel records
- location surveys
- public notices
- in-world artifacts
- Violet-distorted sections

## Appendix and Field Reference

The appendix should lean harder into the emergency binder concept.

Possible treatments:

- tabbed section dividers
- one-page procedures
- laminated-reference visual language
- bold filing codes
- clean tables
- record stamps
- detachable or photocopy-friendly handouts
- lower texture for printer clarity

## Practical Principle

The wiki may feel like a living records system.

The book may feel like the bound master copy from which the town's field materials were assembled.

---

# 15. Things to Avoid

Avoid:

- QuestForge's polished fantasy-dashboard language
- glowing purple borders everywhere
- occult sigils as default decoration
- faux leather book interfaces
- black backgrounds with neon violet text
- brass-and-cog steampunk styling
- constant typewriter typography
- excessive torn paper
- generic tentacles
- Lovecraft silhouettes
- overt monster imagery in navigation
- making every page visibly sinister
- using distortion so often that it loses meaning

The interface should look reliable enough that its failures become frightening.

---

# 16. Final Design Principle

Bellweather's visual identity should not announce horror through ornament.

It should establish trust through civic order, then let the Violet enter through:

- duplicated headings
- altered stamps
- impossible revision dates
- records that know too much
- routes added without authorization
- annotations no clerk remembers writing
- procedures that become too precise
- helpful language that no longer sounds fully human

> The site should feel like Bellweather trying to preserve itself by writing everything down.

> The horror is that something has begun reading along.
